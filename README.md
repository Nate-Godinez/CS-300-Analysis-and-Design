# Data Structures and Algorithms Project

In these projects, I worked on solving the problem of how to store, organize, and display course information for a Computer Science advising system. The goal was to read course data from a file, manage it using appropriate data structures, and then output the courses in alphanumeric order while also allowing users to search for specific course information and view prerequisites. I approached this by first breaking the problem into smaller components, such as loading data, selecting the proper data structure, and implementing sorting and search functionality. I also analyzed how different data structures, including vectors, hash tables, and binary search trees, affect runtime and memory usage. This helped me understand why data structure selection is important, since it directly impacts performance and scalability. One of the main issues I ran into was understanding how to properly evaluate and compare Big-O runtime complexity across different structures, as well as implementing correct sorting and handling course relationships. I passed these challenges by reviewing course materials, testing each function separately, and debugging step-by-step until each component worked as intended.

This project expanded my approach to designing software by encouraging me to think more critically about efficiency and structure before implementation. Instead of focusing only on making the program work, I also considered how design decisions affect performance, maintainability, and scalability. It reinforced the importance of choosing appropriate data structures early in the design process to avoid unnecessary complexity later on. Additionally, it improved the way I write programs by pushing me to focus on readability and organization. I began using clearer function separation, more meaningful variable names, and modular design to make my code easier to understand and maintain. Overall, this project helped me develop a more intentional approach to programming, where I consider both functionality and long-term adaptability when designing software solutions.
